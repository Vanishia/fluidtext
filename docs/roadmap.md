# FluidText Roadmap / 架构规划

目标：做一个“切书的、卡片式阅读”App。核心链路只有一条：**导入 EPUB → 解析章节 HTML → 抽取内容 → 按规则切卡 → 原子写入 Isar → 从库里按顺序读取并展示**。

## 当前范围（MVP）

- 导入 `epub`
- 切分文本并写入数据库（批量写入）
- 主界面按顺序展示卡片


## 现有架构（已实现）

### 分层
- `lib/main.dart`
  - UI：导入按钮 + 导入中阻塞加载层 + 卡片流列表（分页/增量加载）
  - 读取数据库中“最近导入的一本书”的卡片并显示
- `lib/services/`
  - `book_import_service.dart`：EPUB bytes → `epub_plus` 解析 → HTML 转纯文本 → 切分 → `Isar.putAll` 批量入库
  - `text_splitter.dart`：线性“传送带”切分算法（目标字数 + 边界字符落刀）
- `lib/models/`
  - `book.dart`：书（目前只存 `title`、`createdAt`）
  - `book_card.dart`：卡片（`bookId`、`bookTitle`、`cardIndex`、`content`）
- `lib/db/isar_db.dart`
  - 统一打开 Isar 实例（单例）

### 数据库（Isar）
- `Book`
  - `id`（自增）
  - `title`
  - `createdAt`（用于“最近一本书”）
- `BookCard`
  - `id`（自增）
  - `bookId`（关联书）
  - `bookTitle`（冗余字段，方便调试/展示；后续可改为 join 查询或直接用 Book 表）
  - `cardIndex`（顺序编号，从 0 开始递增）
  - `content`（卡片正文）

### 导入流程（当前行为）
1. 选取 `.epub` 文件（`file_picker`，读取 bytes 到内存）
2. `EpubReader.readBook(bytes)` 得到章节列表和 `htmlContent`
3. `html` 包解析 HTML 并取 `document.body?.text` 作为纯文本
4. 对每个章节文本执行切分（默认目标 300 字，遇到 `。！？.!?\n` 等边界落刀）
5. 每章切完后一次性 `putAll` 写入 Isar
6. 首页按 `cardIndex` 升序分页读取并渲染

## 下一阶段架构建议（为了后续功能不返工）

### 1) 书架/多书支持（建议优先级：最高）
动机：现在首页默认加载“最近一本书”，对真实使用不够。
- 新增 `BooksPage`：显示所有已导入书籍
- 点击某本书进入 `CardsPage(bookId)`（复用现在的卡片流）
- 增加“删除一本书”（连同 `BookCard` 一起删）

### 2) 更稳定的导入（建议优先级：高）
动机：真实 EPUB 体积可能很大；导入过程要可感知、可恢复。
- 书籍身份：用 EPUB 文件 hash 和内容指纹识别重复导入
- 原子写入：解析和切卡完成后，再在一个事务中写入 `Book + BookCard`
- 重复导入命中已有书时，不创建新书，直接打开已有 `bookId`
- 基础备份导出：JSON 只作为用户主动备份，不作为主存储
- 导入进度（至少：章节数进度/卡片数进度）
- 导入中禁止二次导入；支持取消
- Android：常驻通知 + 前台服务（防止切书被系统杀掉）
- 更强的容错：空章节、坏 HTML、特殊编码、极长无标点段落

### 3) 为“上下文展开/乱序阅读”补齐必要字段（建议优先级：中）
动机：这俩都是核心玩法，但最好先把数据结构打好地基。
- `BookCard` 增加：
  - `chapterIndex`
  - （可选）`chapterTitle`
- 增加复合索引（概念上）：`(bookId, chapterIndex, cardIndex)`，方便毫秒级查上下文

### 4 乱序阅读实现
基于 bookId 随机抽 cardIndex 或随机查询

### 5 阅读体验（建议优先级：中）
- 卡片加载策略：更明确的分页/缓存策略（目前已做增量加载）
- 字体/行距/主题等（放到最后）

### 6) 图片导入与卡片渲染（建议优先级：中后）
- EPUB 图片资源从 `epub.content.images` 提取，按 hash 落盘到 app documents
- 新增图片资源表，只存 `bookId/hash/href/mimeType/localPath` 等元数据
- 章节 HTML 不再直接 `.text`，而是解析成文本块和图片块组成的内容流
- 第一版图片可单独成卡，后续再做图文混排
- `BookCard.content` 保留纯文本兼容，新增可空 `blocksJson` 描述渲染结构
- 卡片 UI 优先渲染 `blocksJson`，缺失时回退到旧纯文本

#### 为什么第一版图片单独成卡

图片单独成卡不是最终体验一定更好，而是第一版风险最低。

导入时按内容流顺序生成卡片：

`文本块 -> 文本切卡 -> 图片块 -> 图片卡 -> 后续文本块 -> 继续切卡`

这不是在已有卡片中间事后插入，而是在导入草稿阶段顺序递增 `cardIndex`。好处：

- 不破坏现有文本切分器，文字仍按当前规则切
- 渲染简单，图片卡只需要一个图片 block，避免一张卡里同时处理大量文字和大图导致高度、留白、溢出复杂化
- 收藏、已读、上下文仍绑定在 `BookCard` 上，状态模型不需要拆成“文字状态”和“图片状态”
- 旧书兼容最容易：旧卡没有 `blocksJson` 时继续渲染 `content`
- 图文版升级时，图片卡会改变后续 `cardIndex`，所以必须走用户确认的升级流程，不应静默重切旧书

第二阶段再考虑图文混排卡片。适合混排的场景是图片必须贴着前后文字才有意义，但那需要更完整的 block 渲染、卡片高度策略和旧卡状态迁移。

#### 图片导入建议实施顺序

1. 固定文本切分入口：导入时允许用户选择 300/750 字符，避免后续图片上线后再改变文本卡边界
2. 新增内容流中间层：章节 HTML 遍历为 `text/image/break` 等 block，不再直接取 `.text`
3. 新增图片资源落盘：按 hash 保存到书籍资源目录，并建立 `href -> asset` 索引
4. 新增 `BookAsset` 元数据表和 `BookCard.blocksJson`
5. 第一版只生成图片单独卡：图片卡 `content` 可为空或保存 alt，`blocksJson` 只包含 image block
6. 渲染层先判断 `blocksJson`，没有则回退旧 `content`
7. 删除书时同步删除该书图片资源目录
8. 再做旧纯文本书升级为图文版，升级必须保留 `bookId` 并迁移收藏/已读

#### 纯文本旧书升级为图文版

当图片导入上线后，用户可能会重新导入以前导过的同一本书。此时旧书只有纯文本卡片，新导入结果可能包含图片资源和图文渲染结构。

目标行为：

- 检测到同一本书时，不默认创建一个全新副本
- 如果旧书没有图片能力、新导入结果有图片，提示用户“升级为图文版本”
- 升级时保留旧书的 `bookId`，这样书架选择、备注、收藏列表入口不失效
- 用内容指纹把旧卡片状态迁移到新卡片：`isFavorite/favoritedAt/isRead/readAt`
- 图片资源和新卡片在一个事务边界内完成写入；失败时不能留下半升级状态

可行性依赖：

- 图文版导入尽量保持文本切分规则稳定
- 每张文本卡生成归一化内容指纹，用于和旧卡片匹配
- 图片单独成卡的第一版最容易迁移，因为旧文本卡顺序基本不变

迁移策略：

1. 解析新 EPUB，生成图文版导入草稿，不立即写库
2. 根据 `fileHash/contentFingerprint` 找到旧书
3. 如果旧书是纯文本、新草稿包含图片，进入升级流程
4. 对旧卡片按归一化文本指纹建立状态索引
5. 新卡片写入前，根据文本指纹继承旧卡片收藏/已读状态
6. 在事务中删除旧卡片、写入新卡片、更新 `Book` 的图片能力字段
7. 删除或替换旧图片资源目录，避免孤立文件

需要新增的书级字段草案：

- `importVersion`：导入结构版本，例如 `1=纯文本`、`2=图文`
- `hasImages`：当前书是否已经导入图片资源
- `assetImportHash`：图片资源导入指纹，用于判断资源是否需要重建

不自动升级的情况：

- 新旧卡片文本匹配率过低
- 新 EPUB 章节结构差异很大
- 旧书已有较多收藏/已读但无法可靠映射

这些情况应提示用户选择：打开旧书、作为新副本导入、或手动确认升级。

### 功能扩展（建议优先级：低 → 最后）
- 收藏 / 已读
- 筛选模式（只看收藏、过滤已读）
- 全文搜索（Isar 的全文索引 / 额外倒排）

## 建议的实现顺序（可直接按这个做）
1. 书架（多书列表）+ 进入书的卡片页 + 删除书
2. 导入进度与错误提示（UI/状态机）
3. `chapterIndex` 入库 + 上下文查询所需索引/查询方法
4. Android 前台服务/常驻通知（导入防杀）
5. 乱序阅读
6. 收藏/已读/筛选
7. 图片导入与卡片渲染
8. 全文搜索
